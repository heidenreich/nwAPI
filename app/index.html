What exactly is the <NW> API?

The <NW> API is my final project for the Iron Yard Aademy Front End Engineering course. Who is the my in the previous sentence you ask? I am Todd Heidenreich and I have spent endless days and nights in the Next building, while never once taking off my coat. With the guidance of Mason Stewart, I assembled an army of Arduino units (see awesome action photo below) with the sole purpose of detailing the frigid conditions I have endured to achieve coding excellence. With the use of Rick Waldron’s Johnny-Five for NodeJS, and the “StandardFirmata” software package for Arduino, I programmed the Arduino units to collect wether data. These units were placed throughout the Next Innocation Center and began to posting data to a dedicated heroku server.

What's possible?

There are four data points currently collected with Arduino units. With the addition of more sensors the units could collect a wider array of data points.

  • Temperature
  • Light Intensity
  • Location
  • Data Collection Time

On the right is a chart utilizing Chart.JS showing the current temperatures in the NEXT Innovation Center.

Getting started with the <NW> API

Start building an application to let the builing manager know you are very, very cold with actual data collected over time.

The following request:

curl -X GET http://tiny-pizza-server.herokuapp.com/collections/weather

will return a JSON object with the following format:

                      "lightIntensity":"173",
